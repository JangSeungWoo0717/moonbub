## 1. 개요

게임의 뽑기(가챠) 시스템을 **확률·기댓값·시뮬레이션**으로 분석하는 정적 웹 실험실.
"운에 맡기는 뽑기"를 수학적으로 분해해 *기댓값과 위험을 정량화하고, 천장(pity)
시스템 같은 설계 선택이 플레이어 경험에 주는 영향을 실험*한다.

### 핵심 서사 (세특·입시 차별점)
- **수학 역량**: 여사건·기댓값·기하분포·누적확률·몬테카를로 시뮬레이션을 직접 구현하고,
  이론값과 시뮬레이션값이 **큰 수의 법칙**으로 수렴함을 확인.
- **게임학과 역량**: 가챠 확률 밸런싱, 천장 시스템의 수학적 의미, 확률 공정성(천장이
  기댓값과 최악의 경우를 어떻게 바꾸는가)을 분석. 확률을 *게임 설계 도구*로 다룬다.

### 성공 기준
1. 레어도별 확률·천장을 설정하면 목표 획득 **기댓값(기대 뽑기 수)**이 계산된다.
2. "N번 뽑으면 1개 이상 얻을 확률" **누적확률 곡선**이 그려진다.
3. 50%/90%/99% 달성에 필요한 뽑기 수가 표시된다.
4. **몬테카를로 시뮬레이션**으로 분포(히스토그램)·평균±표준편차가 나오고,
   이론 기댓값과 비교된다.
5. 천장 유무를 비교할 수 있다.
6. 실제 뽑기 연출(레어도별 색상·애니메이션)로 게임 느낌을 준다.
7. 모바일·데스크톱 반응형, GitHub Pages에서 서버·키 없이 동작.

## 2. 화면 구성 (모바일 우선, 단일 페이지 + 탭)

- **① 설정 탭** — 레어도 목록(이름·확률·색), 목표 레어도 선택, 천장(pity) 값, 1회 뽑기 비용(선택).
- **② 분석 탭** — 기대 뽑기 수, "N번 안에 얻을 확률" 표, 누적확률 곡선 차트, 50/90/99% 분위 뽑기 수, 천장 적용 기댓값 비교.
- **③ 시뮬레이션 탭** — 반복 횟수 설정 → 몬테카를로 실행 → 히스토그램 분포, 평균/표준편차/최댓값, 이론 vs 시뮬 비교(큰 수의 법칙), 단발 뽑기 연출.
- **④ 방법론 탭** — 여사건·기댓값·기하분포·천장 기댓값 수식, 게임 설계(가챠 밸런싱·천장·공정성) 연계 설명.

## 3. 데이터 모델

```js
Rarity = { name: string, p: number, color: string }   // p: 1회 확률 (0~1)

Config = {
  rarities: Rarity[],     // 예: ★5 0.006, ★4 0.051
  targetIndex: number,    // 분석 대상 레어도 인덱스
  pity: number,           // 천장(이 횟수 안에 목표 보장), 0이면 천장 없음
  costPerDraw: number     // 1회 비용(선택, 기댓값 비용 환산용)
}

SimResult = {
  trials: number[],       // 각 시행에서 목표 획득까지 뽑은 횟수
  mean: number, std: number, min: number, max: number,
  histogram: { bucket: string, count: number }[]
}
```

저장: `localStorage` 키 `gachalab_config` (Config).

## 4. 확률 로직 (직접 구현한 수학 — 방법론 탭 공개)

대상 레어도 1회 확률 `p`, 천장 `H`:

| 개념 | 수식 |
|---|---|
| N번 중 1개 이상 | `1 - (1-p)^N` (여사건) |
| 기대 뽑기 수(천장 없음) | `1/p` (기하분포) |
| q 확률 달성 뽑기 수 | `⌈ ln(1-q) / ln(1-p) ⌉` |
| 천장 적용 기댓값 | `Σ_{k=1}^{H-1} k·(1-p)^{k-1}·p + H·(1-p)^{H-1}` |

모든 함수는 입력값만으로 계산하는 순수 함수 → `node --test`로 검증.

## 5. 시뮬레이션 (몬테카를로 + 큰 수의 법칙)

- 재현 가능한 시드 RNG(mulberry32)로 `Math.random` 비의존 → 테스트 결정론 확보.
- `runs`회 반복하여 각 시행마다 "목표 획득까지 뽑은 횟수"를 기록(천장 적용).
- 평균/표준편차/분포를 계산하고 이론 기댓값과 비교 → 시뮬 평균이 이론값에 수렴함을 시각화.

## 6. 게이미피케이션 (게임학과 연결)

- 레어도별 색상·이모지 연출, 단발/연속 뽑기 애니메이션.
- "운 지수"(이번 시뮬에서 기댓값 대비 빨리/늦게 뽑았는지), 천장 도달 표시.
- 핵심은 *수학적 분석을 게임 밸런싱 의사결정으로 연결*하는 서사.
