# 논리 퀘스트 (ArguQuest) — 설계 명세서

- **작성일**: 2026-06-20
- **과목 / 학년**: 국어 / 고1
- **주제**: 논리적으로 소통하는 힘
- **목표 학과**: 게임학과 (학생부종합전형 세특 활용)
- **배포**: GitHub (Git Publish) → GitHub Pages
- **기술 스택**: 순수 HTML/CSS/JS (빌드 없음), Chart.js(CDN), localStorage

## 1. 개요 (Overview)

토론·논설문의 논증을 **분해 → 시각화 → 점수화 → 결함 점검 → 개선**하는 정적 웹 도구에
**게이미피케이션(레벨·랭크·배지·퀘스트·피드백 연출)**을 입힌 "논리 트레이닝 게임".
딱딱한 논증 학습을 *플레이하며 성장하는 경험*으로 바꾼다.

### 핵심 서사 (세특·입시 차별점)
- **국어 역량**: 툴민(Toulmin) 논증 모델로 주장의 논리 구조를 정밀하게 분해·평가.
- **게임학과 역량**: 점수·보상·피드백으로 이루어진 **게임 메커니즘과 피드백 루프를
  직접 설계**. 플레이어 동기를 끌어내는 랭크/배지/퀘스트 시스템을 만들고, 점수
  밸런싱을 조정하며, MDA(메커닉-다이내믹-에스테틱) 관점으로 학습 경험을 설계했다는
  점이 게임 기획·시스템 설계 역량의 직접 증빙.

### 성공 기준 (Success Criteria)
1. 논증을 6요소로 입력하면 구조 트리가 자동으로 그려진다.
2. 5개 지표 기반 종합 점수(0–100)와 레이더 차트가 표시된다.
3. 점수에 따라 **랭크·레벨**이 부여되고, 조건 충족 시 **배지(업적)**를 획득한다.
4. **퀘스트(미션)** 패널이 다음 개선 행동을 안내한다.
5. 규칙 기반으로 논리적 오류를 자동 경고하고, 수동 체크리스트를 제공한다.
6. 버전을 저장하고 개선 전·후 점수를 비교(성장 그래프)할 수 있다.
7. 점수 상승·배지 획득·레벨업 시 **피드백 연출(애니메이션·토스트)**이 작동한다.
8. 모바일·데스크톱 모두 반응형, GitHub Pages에서 서버·API 키 없이 동작.

## 2. 사용자 흐름 (플레이 루프)

게임의 핵심 루프 = **도전 → 평가 → 보상 → 개선 → 재도전**.

1. **입력(도전)** — 툴민 논증 6요소를 칸에 분해 입력
   `주장(Claim) / 근거(Grounds, 복수) / 전제·보강(Warrant·Backing) / 한정(Qualifier) / 반례·반론(Rebuttal)`
2. **시각화** — 입력을 논증 구조 트리 다이어그램으로 자동 렌더링
3. **평가** — 5개 지표 점수화 → 레이더 차트 + 종합 점수(0–100)
4. **보상** — 점수→랭크/레벨, 조건 충족 시 배지 획득, 피드백 연출
5. **점검** — 논리적 오류 체크리스트(규칙 자동 경고 + 수동 점검)
6. **개선·재도전(퀘스트)** — 퀘스트 안내에 따라 수정 → 저장 → 전·후 성장 비교

## 3. 화면 구성 (모바일 우선, 단일 페이지 + 탭)

- **① 입력 탭** — 논증 6요소 폼, 근거 동적 추가/삭제, 예시 불러오기, **퀘스트 패널**(다음 미션 안내).
- **② 분석 탭** — 트리 다이어그램 · 레이더 차트 · 종합점수 게이지 · **랭크/레벨 배너** · **획득 배지** · 오류 경고 리스트.
- **③ 성장 탭** — 저장된 버전 목록, 두 버전 전·후 점수 비교 막대그래프, 누적 배지 컬렉션.
- **④ 방법론 탭** — 툴민 모델 설명, 점수 산정 기준 공개, **게임 설계(게이미피케이션·피드백 루프·MDA·보상 시스템) 연계 설명**. 세특 발표 근거 자료.

탭 전환은 JS로 섹션 show/hide. 모바일 단일 컬럼 + 탭바.

## 4. 데이터 모델

```js
// 하나의 논증(버전)
Argument = {
  id: string,            // 고유 id (타임스탬프 기반)
  title: string,         // 사용자가 붙인 이름 (예: "사형제 폐지 - 초안")
  createdAt: number,     // 저장 시각
  claim: string,         // 주장
  grounds: string[],     // 근거(복수)
  warrant: string,       // 전제·보강
  qualifier: string,     // 한정 (예: "대체로", "특정 조건에서")
  rebuttal: string,      // 반례·반론 및 그에 대한 대응
  scores: ScoreResult,   // 측정 결과 (저장 시점 스냅샷)
  fallacies: string[],   // 감지/체크된 오류 키 목록
  rank: string,          // 점수 기반 랭크명
  level: number,         // 점수 기반 레벨
  badges: string[]       // 이 버전에서 획득한 배지 키
}

ScoreResult = {
  groundsSufficiency: number, // 0-20
  logicalConnection: number,  // 0-20
  counterHandling: number,    // 0-20
  claimClarity: number,       // 0-20
  fallacyCleanliness: number, // 0-20
  total: number               // 0-100
}

// 전역 진행도 (플레이어 프로필)
Profile = {
  unlockedBadges: string[],   // 누적 획득 배지
  bestScore: number,          // 최고 점수
  totalAttempts: number       // 누적 도전 횟수
}
```

저장: `localStorage` — `argumap_versions`(`Argument[]`), `argumap_profile`(`Profile`).

## 5. 점수 산정 로직 (KPI — 방법론 탭에 투명 공개)

5개 차원, 각 0–20점 → 합산 0–100. 모든 규칙은 클라이언트 측 결정론적 계산.

| 지표 | 측정 방식 (규칙 기반) |
|---|---|
| **근거 충분성** | 근거 개수(2개 이상 가점) + 구체성: 수치/단위/출처 키워드(`%`, `명`, `년`, `통계`, `자료`, `연구`, `에 따르면`) 탐지 |
| **논리 연결성** | 전제·보강(warrant) 작성 여부 및 길이 |
| **반론 대응력** | 반례·반론(rebuttal) 입력 + 대응 표현(`그러나`, `하지만`, `반박`, `그럼에도`) 탐지 |
| **주장 명료성** | 한정어 적절성, 주장 길이, 모호어(`등등`, `뭔가`, `아마`, `것 같다`) 감점 |
| **오류 청결도** | 오류 점검에서 감지된 오류 1건당 감점 (기본 20점에서 차감) |

각 지표 계산 함수는 입력 텍스트와 임계값만으로 점수를 반환(DOM 비의존, 테스트 가능).

## 6. 게이미피케이션 시스템 (게임학과 핵심)

학생이 직접 설계·밸런싱한 보상/피드백 메커니즘.

### 6.1 랭크 & 레벨 (총점 기반)
| 총점 | 랭크 | 레벨 |
|---|---|---|
| 0–40 | 견습 논객 🥉 | Lv.1 |
| 41–60 | 논객 🥈 | Lv.2 |
| 61–80 | 명논객 🥇 | Lv.3 |
| 81–100 | 논리 마스터 💎 | Lv.4 |

### 6.2 배지(업적) — 조건 충족 시 획득
| 배지 | 획득 조건 |
|---|---|
| 근거왕 📚 | 근거 3개 이상 입력 |
| 반론 마스터 🛡️ | 반례·반론 작성 + 대응 표현 포함 |
| 오류 청정 ✨ | 감지된 오류 0건 |
| 퍼펙트 🏆 | 종합 100점 달성 |
| 성장러 📈 | 같은 주제 재도전에서 점수 상승 |

### 6.3 퀘스트(미션) 패널
현재 논증의 약점 지표를 분석해 **다음 행동을 미션으로 제시**
(예: "근거를 1개 더 추가하면 +5점 — 근거왕 배지에 도전!"). 동기 부여 루프 형성.

### 6.4 피드백 연출
점수 게이지 채워지는 애니메이션, 배지 획득 토스트, 레벨업 강조 효과(CSS 트랜지션/키프레임). 과한 의존성 없이 순수 CSS/JS로 구현.

## 7. 오류 점검 규칙 (규칙 기반 자동 + 수동 확인)

| 오류 | 자동 탐지 신호(예시 패턴) |
|---|---|
| 순환논법 | 주장의 핵심어가 근거에 거의 그대로 반복 |
| 성급한 일반화 | `모두`, `항상`, `절대`, `누구나`, `예외 없이` + 근거 부족 |
| 흑백논리 | `~아니면`, `둘 중 하나`, `오직 ~만` 패턴 |
| 허수아비 | 반론 인용이 과장/극단 표현과 결합 |
| 인신공격 | `그 사람은`, `너는`, 인격 비하어 |
| 권위에 호소 | `유명한`, `전문가가 그랬으니` + 근거 부재 |

자동 탐지는 "의심" 경고로 표시, 학생이 체크박스로 최종 판단. 규칙은 `fallacy.js`에
규칙 객체 배열로 분리해 추가/수정 용이.

## 8. 파일 구조

```
index.html                      # 단일 페이지, 4개 탭 섹션
styles/
  main.css                      # 기본 스타일·레이아웃·테마
  responsive.css                # 모바일 우선 미디어쿼리
  game.css                      # 랭크/배지/퀘스트/연출 스타일
js/
  model.js                      # 데이터 모델, 기본값, id 생성
  scoring.js                    # 5개 지표 점수 계산 (순수 함수)
  fallacy.js                    # 오류 탐지 규칙 + 검사 함수
  game.js                       # 랭크·레벨·배지·퀘스트 로직 (순수 함수)
  tree.js                       # 논증 구조 트리 다이어그램 렌더
  charts.js                     # Chart.js 레이더/성장 비교 차트
  storage.js                    # localStorage 저장/불러오기 (버전+프로필)
  app.js                        # 탭 제어, 폼 이벤트, 연출, 전체 조립
data/
  sample-arguments.json         # 예시 논증 1~2개
.github/workflows/deploy.yml    # GitHub Pages 자동 배포
README.md                       # 프로젝트 소개 + 세특 서사 + 사용법
```

각 JS 모듈은 ES Module로 분리, 단일 책임 유지. `scoring.js`/`fallacy.js`/`game.js`는
DOM 비의존 순수 함수로 만들어 테스트·재사용 가능.

## 9. 반응형 / 접근성

- 모바일 우선 CSS. 단일 컬럼(모바일) ↔ 2컬럼(데스크톱).
- 터치 친화적 버튼(최소 44px), 높은 가독성·대비.
- 차트·트리 컨테이너 폭에 맞춰 리사이즈.
- 시맨틱 태그, 라벨-인풋 연결, 키보드 포커스 가시화.

## 10. 배포 (GitHub Pages)

- 빌드 없음 → 정적 파일 그대로 서빙.
